# mock-page

開発およびテスト用のモックページを提供する静的ファイルサーバーです。

## 概要

このサービスは、開発環境でシラバスページのモックを提供するための簡易的な静的ファイルサーバーです。実際の大学のウェブサイトにアクセスせずに、スクレイピング機能のテストや開発を行うことができます。

## 利用技術

- **Node.js** - ランタイム環境
- **静的ファイルサーバー** - HTMLページの配信
- **Docker** - コンテナ化

## ディレクトリ構造

```
mock-page/
├── pages/              # モックHTMLページ
├── Dockerfile.dev      # 開発環境用Dockerfile
└── mimeType.json      # MIMEタイプ設定
```

## 開発環境のセットアップ

### 起動方法

#### Docker Composeを使用する場合

```bash
# プロジェクトルートで実行
docker compose -f docker-compose.yml -f docker-compose.dev.yml up mock-page
```

サービスは`http://localhost:1313`で起動します。

## 動作確認方法

### アクセス確認

```bash
# ホストマシンで実行: モックページが表示されることを確認
curl http://localhost:1313
```

ブラウザで`http://localhost:1313`にアクセスして、モックページが表示されることを確認できます。

## 使用用途

- **フロントエンド開発**: 本番環境にアクセスせずに開発・テストを実行
- **デモンストレーション**: プロジェクトのデモやプレゼンテーション用

## モックページの追加

`pages/`ディレクトリに新しいHTMLファイルを追加することで、新しいモックページを作成できます。

```bash
# 例: 新しいモックページの作成
echo '<html><body>Test Page</body></html>' > pages/test.html
```

## トラブルシューティング

### ポートが既に使用されている

```bash
# ホストマシンで実行: ポート1313を使用しているプロセスを確認
lsof -i :1313

# または別のポートを使用
# docker-compose.dev.ymlのportsセクションを編集
```

### ページが表示されない

- コンテナが正常に起動していることを確認
- `pages/`ディレクトリにファイルが存在することを確認
- ブラウザのキャッシュをクリア

## 注意事項

このサービスは開発環境専用です。本番環境では使用しません。
